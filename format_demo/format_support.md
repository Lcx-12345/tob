# 结论

我可以“生成”非常多种格式；但“是否支持”取决于你希望我对它做什么：

- 纯文本格式：我能直接读写、理解内容并做结构化修改
- 结构化文本格式：我能读写、校验语法、按 schema/约定生成
- 二进制/容器格式：我通常能生成（按公开规范拼装），但无法像纯文本那样可靠地逐行理解与编辑

## 通常强支持（可读可写）

- 文档/标记：.md、.txt、.html、.css、.xml
- 数据：.json、.yaml/.yml、.toml、.csv
- 代码：.py、.js/.ts、.java、.go、.rs、.sh 等
- 矢量图：.svg

## 可生成但不适合“内容级编辑”

- Office OpenXML：.pptx/.docx/.xlsx（本质是 zip + XML 部件）
- PDF、图片（png/jpg/webp 等）：可生成/处理，但更偏“文件级”而非“语义级”编辑

## 示例里的“非强支持”格式

- .zip：我可以生成/解包/打包（文件容器），但它不是可直接阅读的“内容格式”
