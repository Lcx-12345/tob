# 电商商品价格自动化采集与对比工具架构设计

## 1. 系统架构

### 1.1 整体架构

```
┌───────────────────┐      ┌───────────────────┐      ┌───────────────────┐
│                   │      │                   │      │                   │
│   数据采集模块     │──────▶  数据处理模块     │──────▶  数据可视化模块   │
│ (scraper)         │      │ (processor)       │      │ (visualizer)      │
│                   │      │                   │      │                   │
└───────────────────┘      └───────────────────┘      └───────────────────┘
          ▲                          ▲                          ▲
          │                          │                          │
┌───────────────────┐      ┌───────────────────┐      ┌───────────────────┐
│                   │      │                   │      │                   │
│   命令行工具       │      │  数据存储模块     │      │   演示网页        │
│ (cli)             │      │ (storage)         │      │ (web)             │
│                   │      │                   │      │                   │
└───────────────────┘      └───────────────────┘      └───────────────────┘
```

### 1.2 模块说明

1. **数据采集模块 (scraper)**
   - 负责从主流电商平台（京东、淘宝、拼多多）抓取商品信息
   - 支持按关键词搜索采集
   - 包含各平台的具体采集实现

2. **数据处理模块 (processor)**
   - 负责数据清洗、去重和排序
   - 计算性价比指标
   - 生成对比数据

3. **数据可视化模块 (visualizer)**
   - 生成价格趋势图表
   - 商品横向对比图表
   - 性价比推荐标注

4. **数据存储模块 (storage)**
   - 临时存储采集的数据
   - 保存处理结果

5. **命令行工具 (cli)**
   - 提供命令行接口
   - 支持批量采集和参数配置

6. **演示网页 (web)**
   - 提供网页界面
   - 支持输入关键词后现场运行脚本
   - 展示数据示例和结果

## 2. 数据结构设计

### 2.1 商品数据结构

```typescript
interface Product {
  id: string;           // 商品唯一标识
  name: string;         // 商品名称
  price: number;        // 商品价格
  sales: number;        // 销量
  shopScore: number;    // 店铺评分
  url: string;          // 商品链接
  platform: string;     // 电商平台
  image: string;        // 商品图片
  createdAt: string;    // 采集时间
}
```

### 2.2 处理结果数据结构

```typescript
interface ProcessedResult {
  products: Product[];                // 处理后的商品列表
  cheapest: Product;                  // 最低价商品
  averagePrice: number;               // 平均价格
  priceRange: [number, number];       // 价格范围
  recommendations: Product[];         // 性价比推荐
  platformDistribution: Record<string, number>; // 平台分布
}
```

## 3. 核心功能实现

### 3.1 数据采集核心逻辑

1. **请求处理**
   - 使用代理IP避免被封
   - 模拟浏览器请求头
   - 处理反爬机制

2. **页面解析**
   - 使用 XPath 或 CSS 选择器解析页面
   - 提取商品信息
   - 处理动态加载内容

3. **批量采集**
   - 支持多线程/异步采集
   - 控制采集频率
   - 错误处理和重试机制

### 3.2 数据处理核心逻辑

1. **数据清洗**
   - 去除无效数据
   - 统一数据格式
   - 处理缺失值

2. **去重**
   - 基于商品ID或名称去重
   - 保留价格最低的重复商品

3. **排序**
   - 按价格从低到高排序
   - 支持其他排序方式

4. **性价比计算**
   - 基于价格、销量、店铺评分计算
   - 生成性价比评分

### 3.3 数据可视化方案

1. **价格趋势图表**
   - 使用 Chart.js 生成折线图
   - 展示不同平台价格分布

2. **商品横向对比**
   - 生成雷达图或柱状图
   - 对比价格、销量、评分等指标

3. **性价比推荐**
   - 标注高性价比商品
   - 生成推荐理由

## 4. 技术栈选择

### 4.1 后端技术

- **语言**: TypeScript
- **爬虫框架**: Puppeteer (处理动态页面) + Axios (处理静态页面)
- **数据处理**: 原生 TypeScript
- **命令行工具**: Commander.js

### 4.2 前端技术

- **框架**: React
- **样式**: Tailwind CSS
- **图表**: Chart.js
- **状态管理**: Zustand

### 4.3 依赖包

- `puppeteer`: 用于动态页面爬取
- `axios`: 用于HTTP请求
- `cheerio`: 用于HTML解析
- `commander`: 用于命令行工具
- `chart.js`: 用于数据可视化
- `react-chartjs-2`: React的Chart.js封装

## 6. 运行流程

### 6.1 命令行工具运行流程

1. 接收用户输入的关键词和参数
2. 调用数据采集模块获取商品信息
3. 调用数据处理模块处理数据
4. 调用数据可视化模块生成图表
5. 输出结果到终端或文件

### 6.2 网页运行流程

1. 用户在网页输入关键词
2. 点击搜索按钮
3. 后端运行采集脚本
4. 处理数据并生成可视化结果
5. 在网页展示结果

## 7. 性能优化

1. **并行采集**
   - 使用多线程或异步操作并行采集多个平台

2. **缓存机制**
   - 缓存已采集的数据
   - 避免重复请求

3. **延迟控制**
   - 控制请求频率，避免被封

4. **数据处理优化**
   - 使用高效的算法进行数据处理
   - 减少内存使用

## 8. 扩展性

1. **平台扩展**
   - 支持添加新的电商平台
   - 统一的平台接口

2. **功能扩展**
   - 支持更多数据指标
   - 支持更多可视化方式

3. **部署扩展**
   - 支持部署到服务器
   - 支持定时任务

## 9. 注意事项

1. **合法性**
   - 遵守各平台的 robots.txt 规则
   - 避免过度请求影响平台服务

2. **稳定性**
   - 处理各种异常情况
   - 确保系统稳定运行

3. **准确性**
   - 确保数据采集的准确性
   - 定期更新采集规则

4. **安全性**
   - 保护用户输入的关键词
   - 避免存储敏感信息
